## Lessons
- **L12 摄取 API**: `ingestion.ts` handler → auth (API key → project, L09/L10) → Zod-validate a batch
  of events → `processEventBatch` enqueues. Batching; returns fast (L05 "fast lane"). Figures: the
  request lifecycle; a batch envelope. Blocks: table.t (steps), cols (sync vs async), vflow.
- **L13 事件类型与合并语义**: the event envelope deeper; CREATE vs UPDATE; how multiple events for one
  id are destined to merge; idempotency. Figures: event stream → merged record; the type taxonomy.
- **L14 摄取队列**: Redis/BullMQ; primary vs secondary (isolate high-throughput projects); sharding
  (`shardedQueueRegistry`); each event also persisted to S3 (the log). Figures: queue fan-out +
  sharding; S3-as-log.
- **L15 IngestionService**: `mergeAndWrite` — fetch prior events from S3, `mergeRecords` (last-write
  wins by event_ts, mirrors ReplacingMergeTree), per-type processing. The heart of the write path.
  Figures: merge flow; record assembly.
- **L16 Token 计数与成本**: `getGenerationUsage`/`getUsageUnits`; model matching → pricing (Postgres
  Model/Price); provided vs computed (recall L03). tokenisation (`worker/src/features/tokenisation`).
  Figures: provided-wins-computed-backfills; model→price match.
- **L17 ClickhouseWriter**: singleton; batch by table; flush on interval or batchSize; why batching
  (write efficiency); graceful shutdown flush. Figures: batching buffer → flush; per-table queues.
- **L18 OpenTelemetry 摄取**: OTLP endpoint; span→observation mapping; the otel queue; semantics gaps
  vs native. Figures: OTLP span → Langfuse observation mapping.
- **L19 媒体与 blob 存储**: multimodal media handling; S3 media; blob_storage_file_log; how big
  binary payloads stay out of ClickHouse. Figures: media upload + reference; blob log.
