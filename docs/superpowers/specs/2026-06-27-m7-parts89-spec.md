## Part 8 · 仪表盘·指标·成本 / Dashboards, Metrics & Cost (L40–L43)

- **L40 仪表盘/widget 系统 / Dashboards & widgets.** `schema.prisma:1494` Dashboard (definition Json),
  `:1536` DashboardWidget (view, dimensions, metrics, chartType, filters, …). A dashboard = a layout of
  widgets; a widget = a saved query + a chart. Key theme: the widget's query shape (view/filters/metric)
  is the SAME contract L33's Monitor reuses ("mirrors DashboardWidget"). `web/src/features/{dashboard,widgets}/*`.

- **L41 查询引擎 / The query engine.** `packages/shared/src/features/query/*` (dataModel.ts, validateQuery.ts,
  server/). A declarative query (view + dimensions + metrics + filters + time) is compiled to ClickHouse SQL.
  Key theme: a small declarative "query language" → safe parameterized SQL (echoes L23's FilterState→SQL);
  one engine powers dashboards (L40), monitors (L33), and experiment/run aggregation (L35).

- **L42 模型与定价 / Models & pricing.** `worker/src/constants/default-model-prices.json` (seed prices),
  `packages/shared/src/server/pricing-tiers/{matcher,types}.ts`, `web/src/features/models/*`, Prisma `Model`.
  Key theme: how a model is matched (regex/pattern → Model row) and how token usage × price tiers → cost
  (ties back to L16 token/cost). A repo skill `add-model-price` governs this file's edits.

- **L43 云用量计量与花费 / Cloud usage metering & spend.** `worker/src/queues/cloudUsageMeteringQueue.ts`
  (meter tracing events → Stripe billing), `cloudSpendAlertQueue.ts` (alert on spend thresholds). Key theme:
  Cloud-only billing pipeline — periodically count billable events per org and report to a billing provider;
  spend alerts. Frame as Cloud-specific (self-host doesn't bill).

## Part 9 · 自动化与集成 / Automation & Integrations (L44–L47)

- **L44 自动化与 webhook / Automations & webhooks.** `schema.prisma:1635` Trigger, `:1613` Action, `:1659`
  Automation, `:1690` AutomationExecution. `worker/src/queues/webhooks.ts`, `packages/shared/src/server/webhooks/*`
  (validation.ts, **ipBlocking.ts** = SSRF protection). Key theme: an Automation = Trigger (an event, e.g. a
  monitor alert from L33, or a prompt change) → Action (webhook/Slack); webhook delivery must guard against
  SSRF (block internal IPs). This is where L33's monitor alerts get delivered.

- **L45 Slack 与通知 / Slack & notifications.** `web/src/features/slack/*` (OAuth install, channel config),
  `worker/src/queues/notificationQueue.ts`. Key theme: Slack as a first-class delivery channel — encrypted
  workspace connection (SlackIntegration), channel-targeted automations; the notification queue.

- **L46 分析集成 / Analytics integrations.** `worker/src/queues/{postHogIntegration,mixpanelIntegration,
  blobStorageIntegration}Queue.ts`. Key theme: periodically export/stream Langfuse data to external analytics
  (PostHog/Mixpanel) and blob storage (S3/GCS) — "your data, your destinations"; decoupled per-integration queues.

- **L47 批量导出与批量操作 / Batch exports & actions.** `web/src/features/{batch-exports,batch-actions}/*`,
  `worker/src/queues/{batchExport,batchAction}Queue.ts`. Key theme: batch export (stream a big query result to
  a file/blob) and batch actions (apply an operation — delete, add-to-dataset, re-evaluate — to many rows);
  long-running jobs via queues with progress, reusing the L30/L35 job patterns.
